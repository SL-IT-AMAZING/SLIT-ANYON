# 한국 B2B 기업 영업 전략 가이드

> **대상**: AI 앱 빌더를 한국 기업에 판매하는 기술 창업자  
> **목적**: 실전 영업 플레이북 - 첫 미팅부터 계약까지

---

## 1. 첫인상 전략

### 1.1 복장 가이드

**중소/중견기업 미팅**

- 남성: 정장 (넥타이 선택), 구두, 깔끔한 헤어스타일
- 여성: 블라우스 + 슬랙스/스커트, 단정한 메이크업
- ⚠️ 캐주얼 (청바지, 후드티, 운동화)은 리스크. 첫 미팅엔 정장 추천.

**대기업/금융권 미팅**

- 남성: 정장 필수 (네이비/차콜), 넥타이 필수, 구두
- 여성: 정장 상의/하의, 단정한 신발
- ⚠️ 스타트업이어도 대기업 미팅엔 격식 맞추기

**스타트업/IT 기업 미팅**

- 비즈니스 캐주얼 허용 (셔츠 + 면바지, 깔끔한 운동화)
- 단, 첫 미팅엔 정장이 안전

**계절별 팁**

- 여름: 반팔 정장 셔츠도 OK, 땀 주의
- 겨울: 코트 입고 들어가서 벗기 (회의실 들어가기 전)

---

### 1.2 명함 교환 에티켓

**한국 명함 교환 프로토콜 (순서 중요!)**

1. **준비**: 명함은 명함지갑에서 꺼내기 (지갑, 주머니 ❌)
2. **교환 순서**:
   - 방문자(당신)가 먼저 건넴
   - 아래 직급이 먼저 건넴
3. **건네는 방법**:
   - 양손으로 건네기 (한 손 ❌)
   - 상대방이 읽기 쉽게 방향 맞추기
   - 가벼운 허리 숙임 (15도)
   - "안녕하세요, [회사명] [이름]입니다"
4. **받는 방법**:
   - 양손으로 받기
   - 즉시 읽기 (5초 이상)
   - "아, [직함]님이시군요. 만나뵙게 되어 영광입니다"
5. **보관**:
   - 테이블 위에 본인 앞에 놓기 (즉시 넣지 말기)
   - 여러 명이면 좌석 순서대로 배치
   - 미팅 끝나고 명함지갑에 넣기

**NG 행동**

- ❌ 한 손으로 주고받기
- ❌ 명함 위에 뭔가 올려놓기 (커피, 노트북)
- ❌ 명함에 메모하기 (상대방 앞에서)
- ❌ 받자마자 주머니/지갑에 넣기

**명함이 없을 때**

```
"죄송합니다, 명함을 다 소진해서 이메일과 연락처를 문자로 보내드리겠습니다."
→ 즉시 카카오톡/문자로 연락처 전송
```

### 1.3 자기소개 구조 (30초 버전)

**템플릿**:

```
안녕하세요, [회사명]의 [직함] [이름]입니다.
저희는 [구체적인 문제]를 해결하는 [솔루션 한 줄]을 개발하고 있습니다.

[고객사 레퍼런스]에서 [구체적인 성과]를 만들어냈고,
오늘 [상대방 회사]의 [구체적인 페인포인트]에 대해
어떻게 도움을 드릴 수 있을지 말씀드리고 싶습니다.
```

**실전 예시**:

```
안녕하세요, SLIT의 공동창업자 김철수입니다.
저희는 개발자 없이도 AI 앱을 만들 수 있는 노코드 플랫폼을 개발하고 있습니다.

현재 채널톡, 토스뱅크 등에서 내부 AI 툴 개발 시간을 70% 단축하셨고,
오늘 귀사의 내부 업무 자동화에 어떻게 도움을 드릴 수 있을지
말씀드리러 왔습니다.
```

**주의사항**:

- "혁신적인" "세계 최초" 같은 형용사 자제
- 구체적인 숫자와 레퍼런스 강조
- 상대방 회사 상황에 맞춤 (사전 조사 필수)

---

### 1.4 나이가 어릴 때 대응 전략

**한국에서 20대 창업자가 40-50대 임원 만날 때**

**1) 직접 언급하지 말기**

- ❌ "제가 어려서 걱정하실 수 있는데..."
- ✅ 자신감 있게 진행. 나이 언급 안 하면 상대도 안 함.

**2) 실력으로 신뢰 구축**

- 상대방 산업에 대한 깊은 이해 보여주기
- 데이터와 사례로 말하기
- "공부 많이 하셨네요" = 최고의 칭찬

**3) 겸손하지만 비굴하지 않게**

```
❌ "부족하지만 열심히 하겠습니다" (너무 낮춤)
✅ "아직 배울 게 많지만, 이 분야에서는 저희가 가장 깊이 있게 연구했습니다"
```

**4) 팀 강조**

- "저희 CTO는 네이버에서 10년 경력이고..."
- "저희 고문은 삼성전자 전무 출신이십니다"
- 팀의 시니어리티 강조

**5) 레퍼런스 고객 활용**

- "현대자동차 김 상무님도 처음엔 비슷한 우려를 하셨는데..."
- 시니어 고객 사례로 신뢰 전이

---

### 1.5 바디 랭귀지

**DO**:

- ✅ 허리 숙여 인사 (45도 - 첫 인사, 15도 - 일반)
- ✅ 아이컨택 (하지만 너무 오래 쳐다보지 말기)
- ✅ 고개 끄덕이며 경청
- ✅ 메모하기 (관심 표시)
- ✅ 상대방 말 끝날 때까지 기다리기

**DON'T**:

- ❌ 팔짱 끼기 (방어적 인상)
- ❌ 다리 꼬기 (비격식적)
- ❌ 테이블에 팔꿈치 기대기
- ❌ 상대방 말 자르기
- ❌ 너무 큰 제스처 (미국식 손동작)

**한국 특수 상황**:

- 상대방이 전화 받으면 → 조용히 기다리기 (회의실 나가지 말기)
- 담배 제안하면 → 안 피워도 따라가서 대화 (중요!)
- 술자리 제안하면 → 거절하지 말기 (진짜 영업은 술자리에서)

## 2. 한국 비즈니스 문화 핵심

### 2.1 호칭 가이드

**직급 호칭 (정확히 사용하기)**

| 직급             | 호칭                      | 사용 예시               |
| ---------------- | ------------------------- | ----------------------- |
| 사원/주임        | [이름] + 님/씨            | "철수님"                |
| 대리             | 대리님 or [이름] + 대리님 | "김 대리님"             |
| 과장             | 과장님                    | "박 과장님"             |
| 차장             | 차장님                    | "이 차장님"             |
| 부장             | 부장님                    | "최 부장님"             |
| 이사             | 이사님                    | "강 이사님"             |
| 상무/전무/부사장 | [직함]님                  | "김 상무님"             |
| 대표이사         | 대표님                    | "대표님" (성 생략 가능) |

**특수 케이스**:

- 스타트업: "님"만 써도 OK ("철수님")
- 외국계: "매니저님" 가능
- 임원급은 절대 이름만 부르지 말기 (외국계도 마찬가지)

**안전한 호칭법**:
모르겠으면 → "[성] + [추정 직급]님"
예: "김 과장님으로 불러도 될까요?"

---

### 2.2 서열과 좌석 배치

**회의실 좌석 순서**

```
                문
                ↑

    [상석]  [2번]  [3번]
    (임원)  (부장)  (과장)

    ─────────────────
         (테이블)
    ─────────────────

    [당신]  [팀원1] [팀원2]
    (대표)  (팀장)  (팀원)
```

**규칙**:

1. 문에서 가장 먼 자리 = 상석 (임원)
2. 상석 옆 = 2인자
3. 방문자는 문 가까운 쪽
4. 양쪽 팀이 마주보고 앉기

**실전 팁**:

- 먼저 도착하면 하석(문 쪽)에 앉아 있기
- 상대방 팀이 들어오면 일어나서 인사
- "어디 앉을까요?" → "편하신 곳에 앉으세요" → 상대방이 앉은 후 자리 잡기

---

### 2.3 고맥락 문화 (High-Context)

**한국어는 "No"를 직접 말하지 않음**

| 말                        | 실제 의미                   | 대응법                                     |
| ------------------------- | --------------------------- | ------------------------------------------ |
| "검토해보겠습니다"        | 관심 없음 (70%)             | "언제까지 검토 결과 들을 수 있을까요?"     |
| "내부 논의가 필요합니다"  | 거절 (60%)                  | "어떤 부분이 논의 포인트인가요?"           |
| "좋네요"                  | 별로 (50%)                  | "어떤 부분이 가장 관심 가시나요?"          |
| "나중에 연락드리겠습니다" | 거절 (80%)                  | "다음 주 목요일에 제가 연락드려도 될까요?" |
| "비용이 부담되네요"       | 관심 있음! (가격 협상 신호) | "어느 정도 예산을 생각하고 계셨나요?"      |
| "어렵네요"                | 불가능 (90%)                | "어떤 부분이 가장 어려우신가요?"           |

**긍정 신호**:

- ✅ "이거 가능한가요?" (구체적 질문)
- ✅ "우리 팀장님께도 보여드려야겠네요" (상급자 언급)
- ✅ "다음 주에 한 번 더 보시죠" (구체적 다음 미팅)
- ✅ "견적 보내주실 수 있나요?" (진짜 관심)

**거절 신호**:

- ❌ 답변이 모호함
- ❌ 다음 미팅 일정 안 잡힘
- ❌ "좋은데..." 뒤에 침묵
- ❌ 2주 이상 연락 두절

---

### 2.4 눈치 읽기

**미팅 중 신호**:

**관심 있음**:

- 몸을 앞으로 기울임
- 메모를 함
- 구체적인 질문 ("이거 우리 시스템이랑 연동되나요?")
- 동료에게 의견 물음 ("김 과장님, 이거 어떻게 생각해?")

**관심 없음**:

- 시계 자주 봄
- 핸드폰 봄
- "네네" 하며 고개만 끄덕임
- 질문 없음
- 일찍 마무리하려 함 ("오늘 일정이 빡빡해서...")

**미팅 연장 신호** (좋은 신호!):

- "시간 괜찮으세요?" (더 보고 싶다는 뜻)
- "다른 기능도 보여주실 수 있나요?"
- 커피 리필 제안

---

### 2.5 체면 세우기 (Face-Saving)

**절대 하면 안 되는 것**:

- ❌ 상대방을 공개적으로 틀렸다고 하기
- ❌ 상대방 회사/제품 깎아내리기
- ❌ 직속 상사 앞에서 부하 직원과 논쟁
- ❌ "제가 말씀드렸잖아요" (상대방 체면 구김)

**체면 세우는 화법**:

**상대방이 틀렸을 때**:

```
❌ "아니요, 그게 아니라..."
✅ "맞습니다, 그리고 추가로 이런 부분도 있습니다..."
```

**상대방이 이해 못 했을 때**:

```
❌ "이해 못 하신 것 같은데..."
✅ "제가 설명을 명확히 못 드린 것 같습니다. 다시 말씀드리면..."
```

**거절해야 할 때**:

```
❌ "그건 안 됩니다"
✅ "현재 기술로는 어렵지만, 이런 방식으로는 가능합니다"
```

**상대방 제품을 비교할 때**:

```
❌ "[경쟁사] 제품은 느리고 버그 많아요"
✅ "저희는 [구체적 차별점]에 집중했습니다"
```

## 3. 첫 고객 확보 전략

### 3.1 스타트업 → 중견 → 대기업 순서

**왜 스타트업부터?**

- ✅ 의사결정 빠름 (1-2주)
- ✅ 새로운 툴 시도 의향 높음
- ✅ 구매 프로세스 단순
- ✅ 레퍼런스 고객 되어줄 가능성

**타겟 순서**:

```
1단계: 시리즈 A-B 스타트업 (직원 20-100명)
   → 목표: 5-10개 고객 확보, 성공사례 만들기

2단계: 중견기업 (직원 500-2000명)
   → 목표: 1-2개 고객, 대기업 레퍼런스용

3단계: 대기업 (직원 5000명+)
   → 목표: 1개 대형 계약, 브랜드 밸류
```

**스타트업 공략법**:

- Disquiet, GeekNews 커뮤니티 활동
- Product Hunt Korea 런칭
- 스타트업 행사 부스 (COMEUP, NextRise)
- Y Combinator Korea, SparkLabs 데모데이 참석

---

### 3.2 POC (Proof of Concept) 전략

**POC란?**

- 본 계약 전 "진짜 우리한테 맞나?" 검증하는 파일럿 프로젝트
- 기간: 2주-3개월
- 범위: 제한된 기능 + 소수 사용자

**무료 POC vs 유료 POC**

| 구분 | 무료 POC                    | 유료 POC               |
| ---- | --------------------------- | ---------------------- |
| 장점 | 진입장벽 낮음, 많은 기회    | 진심도 높음, 매출 발생 |
| 단점 | 진심도 낮음, 시간 낭비 가능 | 계약 어려움            |
| 추천 | 초기 (첫 10개 고객)         | 레퍼런스 확보 후       |

**POC 성공률 높이는 법**:

**1) 명확한 성공 기준 설정**

```
❌ "잘 되면 계약하겠습니다"
✅ "3가지 조건 충족 시 계약:
   1. 업무 시간 30% 이상 단축
   2. 팀원 80% 이상 만족도
   3. 기존 시스템과 오류 없이 연동"
```

**2) POC 계약서 작성** (간단해도 필수)

```markdown
# POC 계약서

**기간**: 2024년 3월 1일 - 3월 31일 (4주)
**범위**: 마케팅팀 5명, 3개 use case
**성공 기준**:

1. [구체적 KPI]
2. [구체적 KPI]
3. [구체적 KPI]

**POC 후 프로세스**:

- 4월 5일까지 결과 보고
- 성공 시 4월 15일 본 계약 논의
- 본 계약 가격: 월 XXX만원 (연 계약)
```

**3) 주간 체크인 미팅**

- 매주 금요일 30분 미팅
- 진행상황, 이슈, 피드백
- 마지막 주에 갑자기 "안 맞네요" 방지

### 3.3 파일럿 프로그램 프레임워크

**4주 파일럿 템플릿**:

**Week 1: 온보딩**

- Day 1-2: 계정 설정, 교육 세션 (1시간)
- Day 3-5: 첫 use case 만들기 (1:1 지원)

**Week 2: 확장**

- Day 8: 2-3번째 use case 추가
- Day 10: 중간 체크인 미팅 (개선사항 수렴)

**Week 3: 독립 사용**

- 지원 최소화, 스스로 사용하게 하기
- 문제 발생 시 24시간 내 대응

**Week 4: 평가**

### 3.5 한국 성공사례 벤치마킹

**채널톡 (B2B SaaS)**

- 전략: 스타트업 무료 → 중견기업 유료 → 대기업
- 강점: 한국어 고객 지원, 빠른 대응
- 현재: 국내 1위 고객 상담 솔루션

**Sendbird (B2B 채팅 SDK)**

- 전략: 글로벌 먼저 → 한국 역수출
- 강점: 기술력, 해외 레퍼런스
- 현재: 유니콘 (1조원 밸류)

**토스랩 (JANDI, 협업 툴)**

- 전략: 중소기업 → 정부기관/대기업
- 강점: 보안 인증, 한국 시장 이해
- 배운 점: Slack보다 한국 문화에 맞춤

**공통 성공 요인**:

1. 한국어 지원 완벽
2. 빠른 고객 대응 (카톡 상담)
3. 온/오프라인 커뮤니티
4. 무료/저가 시작 → 업셀
5. 한국 기업 문화 이해

### 4.2 발견 (Discovery) (25분)

**목표**: 말하기 < 듣기. 상대방 페인포인트 파악.

**질문 리스트**:

```

"어떤 툴을 사용하고 계신가요?"
"팀 규모는 어느 정도인가요?"
```

**2) 페인포인트**

```
"현재 방식에서 가장 불편한 점은 무엇인가요?"
"어떤 부분에 시간이 가장 많이 드나요?"
```

```
"이 문제를 해결하기 위해 시도해본 게 있나요?"
"다른 솔루션도 검토하셨나요? 어떤 게 있었나요?"
"왜 그 솔루션은 안 맞으셨어요?"
```

**4) 의사결정 프로세스**

```
"이런 툴 도입하실 때 누가 결정하시나요?"
"예산은 어느 부서에서 나가나요?"
"구매 결정까지 보통 얼마나 걸리나요?"
"다른 팀이나 부서 승인도 필요한가요?"
```

**5) 성공 기준**

```

"3개월 후 어떤 변화를 기대하시나요?"
```

- 메모하기 (상대방이 보게)

### 4.3 데모/솔루션 제시 (20분)

**구조**: 문제 요약 → 솔루션 제시 → 데모 → ROI

**1) 문제 요약 (2분)**

```
"말씀 잘 들었습니다. 정리하면,
현재 [문제 1], [문제 2], [문제 3]으로 어려움을 겪고 계시는데,
맞나요?"

[상대방 동의 확인]

"저희 고객사 중 [유사 회사]도 똑같은 문제가 있었습니다."
```

**2) 솔루션 제시 (3분)**

```
"저희 SLIT은 이 문제를 이렇게 해결합니다:

1. [기능 1]: [상대방 문제]를 [방식]으로 해결
2. [기능 2]: [상대방 문제]를 [방식]으로 해결
3. [기능 3]: [상대방 문제]를 [방식]으로 해결

실제로 [레퍼런스 고객]은 이걸로 [성과]를 만들었습니다."
```

**3) 데모 (10-12분)**

- 상대방이 말한 use case로 라이브 시연
- "제가 보여드릴게요 vs 직접 해보시겠어요?" (후자가 좋음)
- 한 가지 플로우만 깊게 (모든 기능 다 보여주지 말기)

**데모 스크립트 예시**:

```
"이사님이 말씀하신 '주간 보고서 자동화'를
직접 만들어보겠습니다.

[화면 공유 시작]

1. 먼저 데이터 소스를 연결합니다 (10초)
2. AI에게 원하는 형식을 말로 설명합니다 (20초)
3. 자동 생성 버튼 클릭 (5초)

보시면 30초 만에 완성됐습니다.
이전엔 얼마나 걸리셨다고 하셨죠? 2시간?
이제 30초입니다."
```

**4) ROI 제시 (3분)**

```
"구체적인 숫자로 말씀드리면:

- 현재: 주 5시간 소요
- SLIT 도입 후: 주 30분
- 절감: 주 4.5시간 = 월 18시간

직원 시급 5만원 기준, 월 90만원 절감
연 1,080만원 절감

저희 연 구독료가 500만원이니,
첫 해부터 580만원 이익입니다."
```

**5) 질의응답 (2분)**

```
"궁금하신 점 있으신가요?"
```

### 5.2 보여줄 것 vs 건너뛸 것

**보여줄 것** (상대방이 말한 use case만):

- ✅ 상대방의 구체적인 문제 해결하는 1-2개 기능
- ✅ 가장 인상적인 "와" 하는 순간 (AI 자동 생성 등)
- ✅ 결과물 (before/after 비교)

**건너뛸 것**:

- ❌ 관리자 설정 (지루함)
- ❌ 모든 메뉴 다 클릭 (혼란)
- ❌ 설치/온보딩 과정 (시간 낭비)
- ❌ 안 물어본 기능 (다음 미팅에 남겨두기)

**데모 흐름**:

```
1. [30초] "이걸 보여드리려고 해요"
2. [3분] 문제 상황 보여주기 (before)
3. [5분] 솔루션 실행 (live)
4. [2분] 결과 확인 (after)
5. [2분] "이걸 당신 팀에 적용하면..." (구체화)
```

### 5.4 라이브 vs 녹화 데모

**라이브 데모**:

- 장점: 상호작용 가능, 신뢰도 높음, 맞춤 가능
- 단점: 실수 가능, 인터넷 의존
- 추천: 대부분의 경우

**녹화 데모**:

- 장점: 완벽한 플로우, 빠름
- 단점: 상호작용 불가, 진정성 낮음
- 추천: 컨퍼런스, 웨비나, 백업용

**하이브리드 (추천)**:

1. 핵심 플로우는 라이브
2. 복잡한 설정 부분은 녹화 (건너뛰기)
3. 백업으로 전체 녹화본 준비

**라이브 데모 팁**:

- 미리 로그인 해두기
- 샘플 데이터 준비
- 핵심 화면에 북마크
- 단축키 사용 (빠르게 보임)

```
이 부분은 다음에 정확히 보여드리고,
일단 다른 기능 먼저 보여드릴게요.

[메모하는 척] 이 부분은 꼭 확인하고 보고드리겠습니다."
```

**3) 로그인 안 됨**

```
"계정 이슈가 있네요.
잠시만요, 다른 계정으로 접속하겠습니다.
[예비 계정으로 로그인]"
```

**4) 느려서 멈춤**

```
"로딩이 조금 걸리네요.
이 기다리는 동안 질문 있으신가요?

[30초 이상 걸리면]
일단 다른 부분 먼저 보여드리고, 이건 나중에 보겠습니다."
```

**예방**:

- 전날 리허설 3번
- 예비 계정 2개
- 녹화 영상 백업
- 핫스팟 데이터 충분히

**마인드셋**:

- 당황하지 않기 (다들 이해함)
- 유머로 넘기기
- 빠르게 플랜 B로 전환
- 절대 변명하지 말기

## 6. 반론 처리

### 6.1 "비싸네요"

**분석**: 진짜 비싸서? vs 가치 못 느껴서?

**대응 1: ROI로 전환**

```
"이해합니다. 가격을 보시면 부담스러우실 수 있어요.
근데 구체적인 숫자로 보시면,

현재 이사님 팀이 이 업무에 주 10시간 쓰시잖아요?
월 40시간, 직원 평균 시급 5만원이면 월 200만원입니다.

저희 솔루션이 월 50만원이니,
월 150만원 절감, 연 1,800만원 이익입니다.

이렇게 보시면 어떠신가요?"
```

**대응 2: 기능 대비 가격**

```
"[경쟁사 A]는 월 80만원이고,
[경쟁사 B]는 월 100만원입니다.
저희는 월 50만원에 [추가 기능들]까지 포함이라
오히려 합리적인 편입니다."
```

**대응 3: 예산 확인**

```
"혹시 어느 정도 예산을 생각하고 계셨나요?"
[금액 들음]

"그 예산이면 [플랜 변경/기능 축소] 방식으로 맞출 수 있습니다."
```

**대응 4: 가치 재확인**

```
"혹시 가격 외에 다른 우려사항은 없으신가요?
가격만 해결되면 진행하실 의향이 있으신 건가요?

[Yes면 협상, No면 다른 이슈 파악]"
```

### 6.2 "지금 시스템으로 충분합니다"

**분석**: 진짜 만족? vs 변화 귀찮아서?

**대응 1: 페인포인트 환기**

```
"현재 시스템 잘 쓰고 계신다니 다행이네요.
그런데 아까 [구체적 불편함]을 말씀하셨잖아요?
그 부분은 어떻게 해결하고 계신가요?"
```

**대응 2: 미래 지향**

```
"지금은 괜찮으실 수 있는데,
팀이 커지면 어떻게 되실 것 같으세요?

저희 고객사 [유사 회사]도 처음엔 엑셀로 충분하다고 하셨는데,
팀이 20명 넘어가니까 한계가 왔다고 하시더라고요."
```

**대응 3: 비교 제안**

```
"그럼 이렇게 하시는 건 어떨까요?
2주 무료 체험해보시고, 현재 시스템이랑 비교해보세요.
더 나은 게 없으면 안 쓰시면 되고요."
```

**대응 4: 은근한 압박**

```
"이해합니다.
참고로 [경쟁사 / 같은 업계 회사]는 이미 저희 솔루션 쓰고 계시는데,
그쪽은 [구체적 성과]를 만드셨어요.

혹시 나중에 관심 생기시면 연락 주세요."
```

### 6.3 "나중에 연락할게요"

**분석**: 거절의 완곡 표현 (80%)

**대응 1: 구체화**

```
"네, 이해합니다.
혹시 '나중'이 언제쯤일까요?
다음 분기? 내년?

[구체적 시점 들음]

그럼 [그 시점 2주 전]에 제가 연락드려도 될까요?"
```

**대응 2: 블로커 확인**

```
"혹시 지금 결정 못 하시는 이유가 따로 있으신가요?
예산? 시기? 아니면 다른 우려사항?

만약 [블로커]가 해결되면 진행하실 건가요?"
```

**대응 3: 작은 약속**

```
"큰 결정은 나중에 하셔도 되고,
일단 제안서만 보내드려도 될까요?
검토하시고 질문 있으시면 연락 주세요."
```

**대응 4: 다른 의사결정자 확인**

```
"혹시 다른 분과도 논의하셔야 하나요?
그분께 제가 직접 설명드릴 수 있으면 좋을 것 같은데,
소개해주실 수 있을까요?"
```

**진짜 나중에 연락할 사람 vs 거절**:

**긍정 신호**:

- 구체적인 시점 제시 ("다음 분기에")
- 이유 설명 ("지금 예산이 없어서")
- 연락처 교환 적극적

**거절 신호**:

- 모호함 ("나중에", "기회되면")
- 이유 없음
- 연락처 안 줌

## 9. 가격 전략

### 9.1 가격 모델 비교

**Per-Seat (사용자당)**

- 구조: 월 1만원 x 사용자 수
- 장점: 확장 가능, 예측 가능
- 단점: 사용자 많으면 비싸짐
- 적합: SaaS, 협업 툴
- 예시: Slack, Notion

**Per-Server (서버당)**

- 구조: 서버/인스턴스당 월 50만원
- 장점: 대규모 사용자에게 유리
- 단점: 복잡함
- 적합: 인프라, DB
- 예시: MongoDB, Redis

**Flat Fee (정액)**

- 구조: 월 100만원 (무제한)
- 장점: 단순, 예측 가능
- 단점: 작은 회사에 비쌈
- 적합: 엔터프라이즈
- 예시: Salesforce Enterprise

**Usage-Based (사용량 기반)**

- 구조: API 호출당 0.01원
- 장점: 공정함
- 단점: 예측 어려움
- 적합: API, AI 서비스
- 예시: OpenAI, AWS

**SLIT 추천 모델** (AI 앱 빌더):

```
Tier 1: Starter (월 30만원)
- 사용자 10명
- 앱 5개
- API 호출 10,000회/월

Tier 2: Professional (월 80만원)
- 사용자 50명
- 앱 20개
- API 호출 50,000회/월
- 전담 지원

Tier 3: Enterprise (협의)
- 무제한
- 커스텀 기능
- 전담 매니저
- SLA 보장
```

**중소기업 (직원 50-200명)**

- 예산: 월 50-200만원
- 결정 기준: ROI, 안정성
- 전략: POC → 부서별 확대
- 계약: 연 계약 (할인 제공)

**중견기업 (직원 200-1000명)**

- 예산: 월 200-500만원
- 결정 기준: 레퍼런스, 기술 검증
- 전략: 파일럿 → 전사 확대
- 계약: 다년 계약 가능

**대기업 (직원 1000명+)**

- 전략: 작은 부서 → 전사 → 계열사
- 계약: 3년 계약, 커스터마이징

### 9.3 POC 가격 전략

**POC 가격 모델**:

**1) 완전 무료**

- 조건: 성공사례 작성 동의
- 기간: 2-4주
- 범위: 제한적 (5명, 3개 use case)
- 추천: 초기 10개 고객

**2) 유료 (저가)**

- 가격: 정가의 10-20%
- 조건: 본 계약 시 환불/크레딧
- 기간: 4-8주
- 범위: 넓음
- 추천: 레퍼런스 확보 후

**3) 유료 (정가)**

- 가격: 정가
- 조건: 계약 없이 POC만 가능
- 기간: 협의
- 범위: 전체
- 추천: 확실한 니즈, 큰 계약 예상

**POC → 계약 전환율**:

- 무료 POC: 20-30%
- 유료 POC: 60-80%

**POC 가격 제안 스크립트**:

```
"POC는 두 가지 옵션이 있습니다:

옵션 1: 4주 무료 POC
- 마케팅팀 5명, 3개 use case
- 대신 성공하시면 사례 작성 협조 부탁드립니다

옵션 2: 8주 유료 POC (월 30만원)
- 전사 확대 가능, 모든 기능
- 본 계약 시 첫 3개월 비용에서 차감

어느 쪽이 더 관심 가시나요?"
```

### 9.4 할인 전략

**언제 할인하면 좋을까?**

**할인해도 되는 경우**:

- ✅ 첫 10개 고객 (레퍼런스 확보)
- ✅ 유명 브랜드 (로고 가치)
- ✅ 큰 계약 (3년 이상, 전사 확대)
- ✅ 연말 예산 소진 시즌
- ✅ 경쟁사와 비교 중 (경쟁 심함)

**할인하면 안 되는 경우**:

- ❌ 상대방이 요청하지 않았는데 먼저 제안
- ❌ 첫 미팅에서 바로 할인
- ❌ 이유 없는 할인 (깎아주면 의심함)
- ❌ 너무 큰 할인 (50% 이상)
- ❌ 모든 고객에게 할인 (기준 없어짐)

**할인 테크닉**:

**1) 조건부 할인**

```
"연 계약 하시면 20% 할인해드립니다"
"5개 팀 이상이면 30% 할인 가능합니다"
"이번 달 안에 계약하시면 첫 2개월 무료입니다"
```

**2) 단계적 할인**

```
❌ "50% 할인해드릴게요" (처음부터)
✅ "10% 할인 가능합니다"
   → "예산이 부족하시다면 15%까지 가능합니다"
   → "최종적으로 20%까지 드릴 수 있습니다"
```

**3) 가치 추가 (할인 대신)**

```
❌ "20% 할인해드릴게요"
✅ "가격은 그대로인데, 대신:
   - 전담 매니저 배정
   - 커스텀 기능 1개 무료
   - 교육 세션 3회 무료"
```

**4) 앵커링 후 할인**

```
"정가는 월 100만원입니다.
하지만 초기 고객사 특별가로 월 70만원에 제안드립니다."
(처음부터 70만원이라고 하면 "비싸다" 느낌)
```

### 9.5 가격 앵커링 기법

**앵커링이란?**

- 첫 번째 숫자가 기준(앵커)이 됨
- 그 다음 숫자는 첫 번째와 비교됨

**테크�ික 1: 높은 플랜 먼저 제시**

```
"저희 플랜은 세 가지입니다:

Enterprise: 월 300만원 (전담 매니저, 커스텀 개발)
Professional: 월 100만원 (50명, 전 기능)
Starter: 월 30만원 (10명, 기본 기능)

어떤 게 관심 가시나요?"
```

→ 100만원이 합리적으로 보임 (300만원 대비)

**테크닉 2: 경쟁사 가격 비교**

```
"[경쟁사 A]는 월 150만원,
[경쟁사 B]는 월 200만원입니다.

저희는 월 100만원에 [추가 기능]까지 포함입니다."
```

→ 100만원이 저렴하게 보임

**테크닉 3: ROI로 프레이밍**

```
❌ "월 100만원입니다"
✅ "이걸로 월 500만원 절감하실 수 있고,
   저희 비용은 월 100만원입니다.
   월 400만원 이익입니다."
```

→ 100만원이 투자로 보임

**테크닉 4: 작게 나누기**

```
❌ "연 1,200만원입니다"
✅ "하루 3만원입니다" (1,200만원 / 365일)
or "직원 1인당 하루 1,000원입니다"
```

→ 작아 보임

**테크닉 5: 디코이 효과**

```
Starter: 월 30만원 (10명)
Professional: 월 80만원 (50명) ⭐️ 인기
Premium: 월 75만원 (30명) ← 디코이

→ Professional이 가성비 좋아 보임
```

**가격 제시 스크립트**:

```
"가격 말씀드리기 전에,
현재 이 업무에 얼마나 비용이 드시나요?

[상대방 답변: 월 500만원]

저희 솔루션은 월 100만원이고,
현재 비용의 80% 절감 가능합니다.

즉, 월 400만원 이익이니,
투자 대비 4배 리턴입니다.

이 정도면 합리적으로 보이시나요?"
```

## 11. 연락 채널 전략

### 11.1 채널별 효과성

**한국 B2B 리드 채널 순위**:

1. **소개 (가장 효과적) 🥇**
   - 전환율: 30-50%
   - 장점: 신뢰 전이, 빠른 미팅
   - 단점: 확장 어려움

2. **행사/컨퍼런스 🥈**
   - 전환율: 10-20%
   - 장점: 대면, 온도 높음
   - 단점: 시즌별, 비용 높음

3. **콜드 이메일 (한글) 🥉**
   - 전환율: 1-3%
   - 장점: 확장 가능
   - 단점: 스팸 느낌, 낮은 응답률

4. **LinkedIn (한국에서 약함)**
   - 전환율: 0.5-1%
   - 장점: 글로벌
   - 단점: 한국에서 잘 안 씀

---

### 11.2 소개 받기 (Warm Intro)

**소개가 최고인 이유**:

- ✅ 신뢰 전이 ("내 친구가 추천한 거면 믿을 만해")
- ✅ 미팅 잡기 쉬움 (거절 어려움)
- ✅ 구매 결정 빠름

**소개 받는 법**:

**1) 기존 고객에게 부탁**

```
"[고객사 담당자]님,
SLIT 잘 사용하고 계신다니 정말 기쁩니다.

혹시 주변에 저희 솔루션이 필요할 만한 분 계신가요?
소개해주시면 정말 감사하겠습니다.

소개해주신 회사가 계약하시면,
[인센티브 / 추가 기능 / 할인] 드리겠습니다."
```

**2) 투자자/멘토에게 부탁**

```
"[투자자]님,
이번 달 목표가 B2B 고객 5개 확보인데,
혹시 소개해주실 만한 기업 있으신가요?

특히 [업종/규모] 기업이면 좋을 것 같습니다."
```

**3) 액셀러레이터 네트워크 활용**

```
"SparkLabs 동기 대표님들께,
저희 SLIT을 소개받을 수 있는 기업 찾고 있습니다.
[간단한 설명]

혹시 소개 가능하신 분 계시면 연락 주세요!"
```

**소개 요청 템플릿**:

```
[소개해줄 사람에게]

"안녕하세요 [이름]님,

[상대방 회사]의 [상대방 이름]님께 저희 SLIT을 소개해주실 수 있을까요?

저희는 [한 줄 소개]이고,
[상대방 회사]의 [구체적 페인포인트]에 도움이 될 것 같습니다.

[레퍼런스 고객]에서 [성과]를 만들었고,
[상대방]님께도 도움이 될 것 같아서요.

이메일 소개나 카톡 단톡방 만들어주시면 감사하겠습니다!

감사합니다 😊"
```

### 11.3 한글 콜드 이메일 템플릿

**콜드 이메일 원칙**:

- 짧게 (5문장 이내)
- 구체적으로 (일반론 ❌)
- 상대방 중심으로 (우리 회사 자랑 ❌)
- 명확한 CTA (다음 액션)

**템플릿 1: 페인포인트 강조**

```
제목: [회사명] [부서] 업무 자동화 제안

안녕하세요, [이름] [직함]님.

SLIT의 김철수입니다.
[회사명]의 [구체적 상황/뉴스]를 보고 연락드립니다.

[업계] 기업들이 [구체적 페인포인트]로 어려움을 겪고 계시는데,
저희는 이 문제를 [솔루션 한 줄]로 해결합니다.

실제로 [유사 회사]는 저희 솔루션으로 [구체적 성과]를 만들었습니다.

다음 주 중 15분 통화 가능하실까요?
[캘린더 링크]

감사합니다.
김철수
```

**템플릿 2: 레퍼런스 강조**

```
제목: 채널톡이 사용하는 AI 앱 빌더 - SLIT

안녕하세요, [이름]님.

채널톡, 토스뱅크, 당근마켓이 사용하는
AI 앱 빌더 SLIT의 김철수입니다.

저희 고객사들은 SLIT으로:
- 개발 시간 70% 단축
- 내부 툴 개발 비용 80% 절감

[회사명]도 관심 있으실 것 같아 연락드렸습니다.

간단한 데모 보여드려도 될까요?
[캘린더 링크]

감사합니다.
```

**템플릿 3: 가치 제안**

```
제목: [회사명] [부서] 생산성 2배 높이는 법

안녕하세요, [이름]님.

SLIT의 김철수입니다.

[업계] 기업들이 [업무]에 주당 10시간을 쓰는데,
저희 고객사는 1시간으로 줄였습니다.

어떻게 가능했을까요?
→ AI 자동화 + 노코드 툴 = SLIT

5분 영상으로 보여드릴 수 있습니다.
[링크]

관심 있으시면 회신 부탁드립니다.

감사합니다.
```

**콜드 이메일 팁**:

- 제목이 80% (열어보게 만들기)
- 1문단 = 1-2문장
- 링크는 1개만
- PS 활용 ("PS: 다음 주 [지역] 방문 예정입니다")

### 11.4 LinkedIn은 한국에서 잘 안 먹힘

**왜 안 먹힐까?**

- 한국은 카톡, 이메일 문화
- LinkedIn 활성 사용자 적음
- B2B도 LinkedIn보다 소개/행사

**LinkedIn 쓸 만한 경우**:

- 외국계 기업 타겟
- 글로벌 진출 준비
- 영어권 고객

**한국 B2B는 이렇게**:

1. 소개 받기 (1순위)
2. 행사에서 명함 받기 (2순위)
3. 한글 이메일 (3순위)
4. 카톡 (가능하면)
5. LinkedIn (마지막 수단)

**카톡 활용법** (연락처 있을 때):

```
"안녕하세요, [이름]님!
[행사/소개]에서 명함 받았던 SLIT 김철수입니다.

혹시 지난번 말씀드린 [내용] 관심 있으실까요?
간단히 자료 보내드려도 될까요? 😊"
```

**한국 B2B 연락 우선순위**:

```
1. 소개 (warm intro) 🔥
2. 행사 명함 → 카톡/이메일
3. 콜드 이메일 (한글)
4. 전화 (사전 이메일 후)
5. LinkedIn (외국계만)
```

## 결론: 한국 B2B 영업의 핵심

### 🎯 3가지 핵심 원칙

**1. 관계 > 제품**

- 한국은 관계 기반 사회
- 첫 미팅에서 계약 기대 ❌
- 신뢰 쌓기 → 소개받기 → 확장

**2. 인내심**

- 대기업 계약: 6-12개월
- 포기하지 말고 꾸준히 팔로업
- "나중에 연락할게요" = 거절 아닐 수도

**3. 현지화**

- 한글 완벽히
- 한국 비즈니스 에티켓 준수
- 한국 문화 이해 (고맥락, 서열, 체면)
