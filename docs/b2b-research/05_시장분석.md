## 2. 바이브 코딩 트렌드

### 2.1 정의

"바이브 코딩(Vibe Coding)"은 Andrej Karpathy(전 Tesla AI 디렉터, OpenAI 공동창업자)가 2025년 2월에 제안한 개념. 자연어로 의도를 설명하면 AI가 코드를 생성하는 방식. 기존 코딩과의 핵심 차이:

| 구분      | 전통적 코딩               | 바이브 코딩                |
| --------- | ------------------------- | -------------------------- |
| 입력      | 프로그래밍 언어           | 자연어 (한국어/영어)       |
| 필요 역량 | 문법, 알고리즘, 디버깅    | 문제 정의, 요구사항 명확화 |
| 속도      | 수일~수주                 | 수분~수시간                |
| 대상      | 전문 개발자               | 모든 직원 (시티즌 개발자)  |
| 반복      | 코드 수정 → 빌드 → 테스트 | 프롬프트 수정 → 즉시 확인  |

### 2.2 주요 도구별 한국 채택 현황

| 도구               | 유형         | 월간 사용자 (글로벌) | 한국 추정 사용자 | 가격       | 한국어 지원 |
| ------------------ | ------------ | -------------------- | ---------------- | ---------- | ----------- |
| GitHub Copilot     | IDE 플러그인 | 1,500만+             | 30만+            | $10-19/월  | 부분        |

| ChatGPT (OpenAI)   | 대화형 AI    | 2억+                 | 800만+           | $20/월     | 양호        |

### 2.3 한국 시장 사례

**패스트캠퍼스 바이브 코딩 강의**

- 2025년 출시 후 수강생 1만 명 돌파
- 매출 약 12억원 (수강료 평균 12만원 기준)
- 수강생 구성: 비개발자 60%, 주니어 개발자 25%, 시니어 개발자 15%
- 주요 수강 동기: "사이드 프로젝트를 혼자 만들고 싶다", "업무 자동화 도구를 직접 만들고 싶다"

**비개발자 앱 개발 사례**

- 마케터가 Cursor로 고객 리드 관리 대시보드 제작 (개발 시간: 3시간)
- 인사팀 직원이 ChatGPT + Bolt.new로 채용 현황 트래커 제작
- 영업 담당자가 Claude로 CRM 연동 자동 보고서 생성 도구 제작
- 공장 관리자가 Replit으로 생산 라인 모니터링 대시보드 제작

### 2.4 한국 미디어/인플루언서 동향

**주요 보도**:

- 조선일보: "코딩 몰라도 앱 만든다... '바이브 코딩' 열풍" (2025.03)
- 매일경제: "AI가 개발자 대체? 바이브 코딩의 가능성과 한계" (2025.04)
- 한국경제: "개발자 없이 사내 앱 만드는 시대" (2025.05)

**유튜브/블로그 인플루언서**:

- 노마드코더 (95만 구독): 바이브 코딩 시리즈 100만뷰+
- 조코딩 (65만 구독): "비개발자 앱 만들기" 시리즈
- 생활코딩 (120만 구독): AI 코딩 도구 비교 리뷰
- 드림코딩 (48만 구독): Cursor/Copilot 실전 가이드

### 2.5 정부 입장

- **과학기술정보통신부**: "AI 활용 소프트웨어 인력 양성" 정책 추진
- **디지털플랫폼정부위원회**: 공공기관 AI 도구 도입 권장
- **중소벤처기업부**: 중소기업 AI 도입 지원 바우처 프로그램 운영 (기업당 최대 3억원)
- 전반적으로 **긍정적 기조**이나 "AI 생성 코드의 보안 검증" 이슈 제기 중

### 2.6 시사점 — 우리 제품에 대한 의미

바이브 코딩의 폭발적 성장은 Dyad에 **직접적 시장 기회**:

1. "바이브 코딩으로 앱을 만들었는데, 사내에서 어떻게 배포하지?" → **우리가 답**
2. 비개발자도 앱을 만들 수 있게 됨 → **잠재 사용자 10배 확대**
3. 망분리 환경에서는 Bolt.new, v0 사용 불가 → **온프레미스 바이브 코딩 = 우리만 가능**

### 3.6 데이터 프라이버시 처리 방식

한국 기업이 사내 AI 데이터 보안을 처리하는 3가지 패턴:

**패턴 1: 클라우드 + 데이터 마스킹 (40%)**

- Azure OpenAI + PII 필터링
- 장점: 빠른 도입, 최신 모델 사용
- 단점: 완전한 보안 보장 불가, 규제 산업 부적합

**패턴 2: 프라이빗 클라우드 (35%)**

- 네이버클라우드/KT클라우드 + 전용 인스턴스
- 장점: 국내 데이터 센터, 준수한 보안
- 단점: 비용 높음, 모델 선택지 제한

**패턴 3: 완전 온프레미스 (25%)**

- Ollama/vLLM + 자체 서버
- 장점: 완전한 데이터 통제, 망분리 호환
- 단점: GPU 투자 필요, 모델 성능 제한

**시사점**: 패턴 3이 우리 타겟. 25%이지만 금융/공공/방산 = 고가치 고객.

## 4. 시티즌 개발자 운동

### 4.1 정의

시티즌 개발자(Citizen Developer) = IT 부서 소속이 아닌 일반 직원이 로우코드/노코드 도구를 사용하여 직접 앱이나 자동화를 만드는 것.

### 4.2 한국 vs 글로벌 현황

| 지표                                | 글로벌 | 한국 | 갭               |
| ----------------------------------- | ------ | ---- | ---------------- |
| 적극 채택 기업 비율                 | 35%    | 10%  | -25%p            |
| 시티즌 개발자 비율 (전체 직원 대비) | 8%     | 2%   | -6%p             |

### 4.5 IT 부서의 역할과 거버넌스

한국 기업에서 시티즌 개발자 프로그램 성공의 핵심은 **IT 부서의 역할 전환**:

| 기존 역할              | 새로운 역할            |
| ---------------------- | ---------------------- |
| 모든 앱 직접 개발      | 플랫폼 제공 + 거버넌스 |
| 요청 → 대기 → 개발     | 셀프서비스 환경 구축   |
| 게이트키퍼 (막는 역할) | 인에이블러 (돕는 역할) |
| 코드 리뷰              | 템플릿/가이드라인 제공 |

**거버넌스 모델 (Best Practice)**:

1. **승인된 도구만 사용** — IT에서 검증한 플랫폼만 허용
2. **데이터 접근 권한 관리** — 민감 데이터 접근은 IT 승인 필요
3. **앱 등록/감사** — 만든 앱은 반드시 IT에 등록
4. **교육 필수** — 보안/컴플라이언스 교육 이수 후 사용
5. **정기 리뷰** — 분기별 사용 현황 점검

### 6.3 성공 기업의 5가지 공통점

1. **작게 시작, 빠르게 확산**: 1개 부서 1개 유스케이스로 시작 → 성공 후 확대
2. **명확한 KPI 설정**: "응답 시간 50% 단축", "수동 작업 30시간/월 절감" 등 측정 가능한 목표
3. **경영진 스폰서**: C-level이 직접 챔피언으로 참여
4. **내부 역량 확보**: 외주 대신 내부 AI 팀 구축
5. **거버넌스 선행**: 보안/컴플라이언스 프레임워크를 프로젝트 시작 전에 수립

### 6.4 시사점 — Dyad의 기회

PoC 피로감은 Dyad에게 **역설적 기회**:

- "PoC를 빠르게 만들 수 있다" → 실패 비용 최소화
- "비개발자도 만들 수 있다" → 인력 부족 해결
- "온프레미스로 바로 배포" → 보안/컴플라이언스 해결
- "Docker 기반이라 운영 부담 최소" → 유지보수 비용 절감

**영업 메시지**: "88%가 실패하는 이유를 우리가 해결합니다"
